# Edge-Gesture Hub 硬件连接文档（代码对齐版）

## 1. 接线总表

| 模块 | 接口 | STM32 引脚 | 备注 |
|---|---|---|---|
| MPU6050 | I2C1 | PB6(SCL), PB7(SDA) | 地址常用 0x68 |
| OLED(SSD1306/SH1106兼容) | I2C2 | PB10(SCL), PB11(SDA) | 地址常用 0x3C/0x3D |
| USB-TTL 调试 | USART1 | PA9(TX), PA10(RX) | 115200 8N1 |
| 无线串口 | USART2 | PA2(TX), PA3(RX) | 9600 8N1 |
| 业务 LED | GPIO | PD12 | ANOM 边沿触发切换 |

## 2. 电气建议
- 全系统共地。
- I2C 线路建议保留上拉（模块已带上拉可不重复加）。
- 外接 LED 建议串联限流电阻（220R~1k）。

## 3. 与当前代码一致的要点
- MPU6050 与 OLED 是双 I2C 总线，不是单总线共线。
- 串口是双通道：一个调试、一个无线。
- OLED 由 `hi2c2` 驱动，MPU6050 由 `hi2c1` 驱动。

## 4. 联调检查
- I2C 设备在线：MPU6050 与 OLED 均可初始化。
- OLED 显示：`STATUS:` 与 `LEARN/NORM/ANOM`。
- 串口数据：连续输出 `ax,ay,az,gx,gy,gz,cls`。
- 动作触发：出现 `ANOM` 边沿时 `PD12` 切换。

## 5. 旧口径废弃声明
以下说法不再适用于当前代码：
- OLED 与 MPU6050 共用 `PB6/PB7`。
- 仅使用 `USART1 115200` 输出全部数据。
- 使用 JustFloat 作为当前协议。
